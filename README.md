# 個人タクシー向け 営業CSV・売上・経費管理Webデモ

個人タクシー事業者向けに、日々の営業CSVをもとに売上、決済、経費、未入金、差額あり、月次確認、確定申告向け集計までをまとめて確認するための商談・ヒアリング用フロントデモです。

法人タクシー会社の管理部門向けではなく、個人タクシー事業者が毎日の事務作業を減らすWebツールとして見せられるよう、画面文言とサンプルデータを調整しています。

## 現在の状態

- Next.js App Router / React / TypeScript / Tailwind CSSで実装
- DB、ログイン、サーバー保存、API連携は未実装
- データは `src/lib/demoData.ts` の固定デモデータを使用
- 商談・ヒアリング時に画面イメージ、必要項目、本実装範囲を確認するためのフロントデモ
- 画面上に「デモ版」バッジと、本実装ではCSV/DB保存対応予定である注記を表示
- PC表示を優先し、タブレット幅でも横スクロールや折り返しで崩れにくい構成

## ターゲット

- 個人タクシー事業者
- 1人または少人数で、営業CSV、売上、決済、経費、未入金状況をまとめて確認したい事業者
- 月次確認や確定申告準備のために、日々の営業データと事業用経費を整理したい事業者

## 主な機能

- ダッシュボード: 個人タクシー向けサービス概要、利用フロー、本日売上、今月売上、営業回数、キャッシュレス比率、経費合計、差引利益、最近取り込んだCSV、決済内訳、月間売上推移
- CSV取込デモ: ファイル選択、ファイル名表示、LT27 CSVのF1/F2/F3/F4/FA説明、取込プレビュー、解析結果風カード、取込済み一覧、完了メッセージ
- カード読込デモ: タクシーメーターSDカード生データを複数選択し、ファイル名、拡張子、サイズ、ArrayBuffer読込結果をブラウザ上で確認。.TF4 は仕様書ベースに寄せた営業明細・日報プレビュー、.TFA/.SPY は決済種別候補と突合プレビュー、TF4合計金額を基準にした現収/未収/立替の集計サマリーを表示。日報プレビュー結果はメモリ上の一時デモデータとして日報・決済集計画面へ反映可能
- 日報: URLクエリ対応、日付切り替え、未登録日の空状態、CSV未取込日の専用表示、帳票風ヘッダー、CSV上の乗務員コード、CSV上の車両コード、出庫/入庫時刻、営業回数、売上、決済内訳、経費、利益、営業明細、CSV/PDF/印刷ボタン
- カレンダー: 日報への日付リンク、月間カレンダー、売上、経費、利益、CSV未取込日、営業なし日、売上高めの日、週/月/前月比サマリー
- 決済集計: 日次、週次、月次、期間指定の切り替え、決済種別別の金額、件数、構成比、バー表示、決済種別クリック時の利用明細
- 決済突合: 突合済み、差額あり、未入金のステータス編集、備考編集、すべて/突合済み/差額あり/未入金/未入金・差額ありの絞り込み
- 経費管理: 今月経費、ガソリン代、車両関連費、通信・その他、件数、領収書未添付件数のサマリー、年月を明示した経費内訳、一覧、表示月を切り替えられる経費カレンダー、日別経費と選択日の明細、月次集計、月別の項目別集計、ラベル付き追加フォーム、画面上だけの一時追加、領収書画像アップロードボタン
- 設定: 事業者情報、本人・ドライバー情報、車両情報、決済種別マッピング、QR/NETはデモ用分類である注記、CSV取込ルール

## デモデータ

- 日報デモデータは `2026-06-24` から `2026-06-30` までの1週間分
- 経費デモデータは2026年6月を中心に、月次集計用として2026年5月、2026年4月のサンプルも含む
- 経費カレンダーは2026年4月、5月、6月の月選択に加え、前月/今月/翌月でデータがない月も表示可能
- 経費管理では表示月ごとに、カテゴリ別の合計金額、件数、構成比、簡易バーを表示
- 正式版では実日付の現在月を初期表示の基準にする想定。本デモでは2026年6月を基準月として表示
- 通常営業日、売上が高い日、経費が多い日、営業なし日、CSV未取込日を含む
- カレンダー、日報、決済集計、経費管理で同じ日付・金額感になるように調整
- サンプル事業者は `山田個人タクシー`
- 管理コードは `TAXI-001`
- 基本は本人1名、事業用車両1台の見せ方

- タクシーメーターSDカード生データの本格解析、保存、日報・決済集計への反映
- 乗車地・降車地の住所取得精度が必要な場合は、Google Maps API等への切り替えを検討
- 領収書画像保存
- 経費の月次/年次集計
- DB保存済み経費をもとにした過去月確認、月次集計、申告向け集計
- 確定申告向け経費出力
- 日報PDF/CSV出力
- 決済集計CSV出力
- 決済入金データとの突合

## デモで保存されないもの

- CSVファイル
- カード読込で選択したSDカード生データファイル
- カード読込から日報・決済集計へ反映した一時デモデータ
- 経費追加
- 決済明細の突合ステータス
- 決済明細の備考
- 設定内容
- PDF/CSV出力ファイル
- 領収書画像

## ダミー実装

- CSV解析、カード生データ解析、ファイル保存、サーバー保存は行いません。
- カード読込はブラウザ上でArrayBuffer読込可否、.TF4の営業明細・日報プレビュー、.TFA/.SPYの決済種別候補との突合プレビュー、TF4合計金額ベースの現収/未収/立替集計サマリーを確認するだけで、元ファイルの変更、本番DB保存、サーバー送信、既存画面への反映は行いません。
- 「デモデータとして反映」はReact Context上のメモリ一時状態のみを使います。localStorage、IndexedDB、API送信、DB保存は行わず、リロードすると消えます。
- カード読込の日報プレビューでは、「乗降地名を取得」ボタン押下時のみ、TF4の乗車地GPS・降車地GPSをもとにNominatim/OpenStreetMapで地名を表示します。DGP等の走行軌跡は送信しません。住所取得はデモ用で、精密な住所保証は行いません。
- 日報CSV出力、PDF出力、印刷、決済CSV出力、領収書画像アップロード、設定保存は `alert` のみです。
- 経費追加はブラウザ上の一時的な状態反映のみで、表示月の経費内訳、一覧、表示月に応じたカレンダー、月次集計、月別の項目別集計、サマリーカードへ反映されます。リロードすると戻ります。
- 決済明細の突合ステータス変更、備考編集、ステータス絞り込みはブラウザ上の一時状態のみで、リロードすると戻ります。
- 日付切り替えは固定デモデータの範囲でのみ表示が変わります。CSV未取込日は専用表示、範囲外の未登録日は空状態を表示します。

## カード読込デモ確認手順

1. 開発サーバーを起動します。

```bash
npm run dev
```

2. ブラウザで `/card-import` を開きます。
3. サンプルの `.TF4` / `.TFA` / `.SPY` ファイルをまとめて選択します。
4. 日報プレビューで、営業明細、総営収、現収、未収、総営収との差額、決済種別を確認します。
5. 必要に応じて「乗降地名を取得」ボタンを押し、乗車地・降車地の地名表示を確認します。
6. 「デモデータとして反映」ボタンを押します。
7. `/daily-report` を開き、カード読込データから作成した日報プレビューを確認します。
8. `/payments` を開き、カード読込データから作成した決済集計を確認します。

注意事項:

- 乗降地名の取得対象は、TF4営業明細の乗車地GPS・降車地GPSのみです。
- DGP等の走行軌跡データやその他GPSデータは地名取得サービスへ送信しません。
- 地名取得はデモ用です。住所精度は保証せず、本番ではGoogle Maps API等への切り替えを想定しています。
- カード読込結果と「デモデータとして反映」した内容は、メモリ上の一時データです。DB保存、localStorage、IndexedDB保存は行いません。

## 逆ジオコーディング

カード読込の日報プレビューで乗車地・降車地の地名を表示する場合は、`src/app/api/reverse-geocode/route.ts` のサーバー側API RouteからNominatim/OpenStreetMapへ問い合わせます。APIキーは不要です。

地名取得はファイル読込直後には自動実行せず、画面上の「乗降地名を取得」ボタン押下時のみ実行します。取得対象はTF4営業明細の乗車地GPS・降車地GPSのみで、DGP等の走行軌跡やその他GPSデータは送信しません。

クライアントからNominatimへ直接アクセスせず、画面内キャッシュで同一座標への問い合わせを抑えます。Nominatimはデモ用途の確認用です。本番では利用条件、精度、アクセス量を考慮し、Google Maps API等への切り替えを検討します。
